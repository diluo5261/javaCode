
ch06-path :解决jsp页面中的路径问题,用 /  和 不用 /

在jsp,html中使用的地址,都是在前端页面中的地址,都是相对地址

地址分类
1. 绝对地址,带有协议的名称的是绝对地址
2. 相对地址,没有协议开头的,都是相对地址  例如:user/some.do
    相对地址不能独立使用,必须有一个参考地址,通过参考地址+相对地址本身才能指定资源

3.参考地址
    在你的页面中,访问地址不加 /
    当你的地址没有 / 开头,当你点击连接时,访问地址时当前页面的地址,加上连接的地址

    在你的页面中,访问地址加 /
    / 被解析成 协议 :// ip地址 :端口号/


    解决方案:加入${pageContext.request.contextPath}
